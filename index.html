ğŸ“‹ Deskripsi Studi Kasus

Dalam pemrograman, seringkali kita perlu menganalisis teks untuk berbagai keperluan seperti validasi input, pemrosesan dokumen, atau analisis data. Salah satu tugas sederhana namun fundamental adalah menghitung jumlah huruf kapital dalam sebuah string.

Studi kasus ini membandingkan dua pendekatan algoritma untuk menyelesaikan masalah tersebut:

  • Algoritma Iteratif - Menggunakan perulangan (loop) untuk memeriksa setiap karakter
  • Algoritma Rekursif - Menggunakan pemanggilan fungsi diri sendiri secara berulang

Melalui analisa ini, kita akan memahami perbedaan performa dan efisiensi memori dari kedua pendekatan tersebut.

ğŸ”„ Metode Iteratif Looping

Jumlah Kapital: -
Waktu Eksekusi: - ms
Kompleksitas Waktu: O(n)
Kompleksitas Ruang: O(1) 
function countCapitalIterative(str) {
  let count = 0;
  for (let i = 0; i < str.length; i++) {
    if (str[i] >= 'A' && str[i] <= 'Z') {
      count++;
    }
  }
  return count;
}

ğŸ” Metode Rekursif Self-Calling

Jumlah Kapital: -
Waktu Eksekusi: - ms
Kompleksitas Waktu: O(n)
Kompleksitas Ruang: O(n) 
function countCapitalRecursive(str, index = 0) {
  // Base Case
  if (index >= str.length) return 0;
  
  // Check current character
  const isCap = (str[index] >= 'A' 
              && str[index] <= 'Z') ? 1 : 0;
              
  // Recursive Step
  return isCap + countCapitalRecursive(str, index + 1);
}

ğŸ“Œ Kesimpulan Memori:
Meskipun kedua algoritma memeriksa setiap karakter sekali (Linear Time O(n)), Metode Iteratif jauh lebih baik untuk teks panjang karena kompleksitas ruang O(1).

Metode Rekursif memiliki kompleksitas ruang O(n). Jika input sangat panjang (> 8.000 karakter), metode rekursif akan menyebabkan Stack Overflow (error), sedangkan iteratif tetap berjalan lancar.

ğŸ† Kesimpulan Akhir

Untuk studi kasus "Menghitung Huruf Kapital", algoritma Iteratif adalah pilihan yang lebih baik karena:

  • Penggunaan memori yang konstan O(1) dibanding O(n)
  • Tidak ada risiko Stack Overflow untuk input besar
  • Performa yang lebih konsisten dan predictable
  • Overhead yang lebih rendah tanpa recursive function calls

Namun, algoritma Rekursif tetap valuable untuk dipelajari karena memberikan pemahaman tentang konsep divide-and-conquer dan pemecahan masalah secara matematis.
